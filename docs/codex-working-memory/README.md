# Codex Working Memory

This directory vendors the local long-term notes from the main Codex session so
remote Codex instances can pick up the same project context.

Start here:

1. `../remote-codex-handoff.md` for the short operational handoff.
2. `research-log.md` for the detailed implementation/research chronology.
3. `backlog.md` for unresolved prototype work.
4. `progress-log.md` for earlier checkpoint history.
5. `workflow-spine-spec.md` for the intended closed-loop workflow structure.
6. `tochi-writing-synthesis.md` for paper-framing notes.

These files are intentionally imperfect working notes. Treat them as context,
not polished documentation. If remote work changes direction, update these notes
or append a new dated section rather than relying only on chat history.
